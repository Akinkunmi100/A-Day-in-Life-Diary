# Unified Master Codebook — 11 Families (A–K)
# Telecom Ethnography: "A Day in the Life" Diary Study
# 11 respondents, Lagos, Nigeria

## How to Use This Codebook

- **Apply codes at the segment level**: each diary response (answer to a question) is a coding unit
- **Multi-code**: a single response can receive multiple codes from different families
- **Code what is present, not what you infer**: if a respondent doesn't mention network issues, don't code B02
- **"Nothing" responses**: code as N00 only if truly empty of content. "Nothing" as a response to "What slowed you down?" = no stress = analytically meaningful, not N00

## Family A: Work & Productivity

| Code | Label | When to Apply |
|:---:|:---|:---|
| A01 | Phone-as-Business-Tool | Phone used directly for work: calls to clients, WhatsApp commerce, sending designs, checking orders |
| A02 | Offline Fallback Strategy | Respondent describes what they'd do without phone: handbills, walking to client, using someone else's phone |
| A03 | Remote/Hybrid Work | Working from home or outside the office; submitting tasks digitally |
| A04 | WhatsApp Commerce | WhatsApp specifically used for business: advertising, customer contact, order management |
| A05 | Workplace Activity | Describes office/shop/market tasks: attending training, teaching, helping colleagues |
| A06 | Professional Identity | Describes self through work role: teacher, trader, designer, insurance agent |

## Family B: Network Experience

| Code | Label | When to Apply |
|:---:|:---|:---|
| B01 | Network Loyalty Rationale | Explains why they use a specific network: "that's the number people know me with", "bundle is affordable" |
| B02 | Network Pain Point | Poor network, dropped calls, slow data, "bad network" as a complaint |
| B03 | App/Digital Payment Failure | Banking app fails due to network: "GTB app making transfer network was bad" |
| B04 | Blocked Goal (Network-Caused) | Something they wanted to do was prevented by network: "couldn't book a ride", "couldn't watch movie" |
| B05 | Transfer/Transaction Failure | Specific failed financial transaction due to network |
| B06 | Multi-SIM Behaviour | Uses different networks for different purposes: GLO for data, MTN for calls |
| B07 | Data-vs-Call Trade-off | Explicit mention of choosing between data and voice based on cost or availability |
| B08 | Network Satisfaction | Positive statement about network: "all was smooth", "network was good today" |

## Family C: Mobility & Movement

| Code | Label | When to Apply |
|:---:|:---|:---|
| C01 | Commute Stress | Stress from transportation: traffic, bad roads, noise, crowding |
| C02 | Transport Mode | Named transport type: bus, keke/tricycle, okada/bike, BRT, ride-hail app, walk |
| C03 | Mobility Constraint | Unable to travel as planned: curfew, cost, safety, availability |
| C04 | Phone-While-Mobile | Using phone during transit: music, messages, transfers |
| C05 | Area/Location Mention | Named Lagos areas (Surulere, Ojuwoye, Ogba, Yaba) as mobility context |
| C06 | Weather/Environmental Disruption | Rain, flooding, bad roads, erosion affecting movement or commerce |

## Family D: Financial Behaviour

| Code | Label | When to Apply |
|:---:|:---|:---|
| D01 | Necessity Spending | Spending framed as unavoidable: food, transport, data, bills |
| D02 | Budget Constraint | Explicit mention of limited money: "no money", "sales was poor", budget considerations |
| D03 | Spending Alternatives Considered | Respondent describes options weighed before spending: "should I save or spend?" |

| D05 | Cash Preference | Preference for or defaulting to cash over digital payment |
| D06 | Impulse/Unplanned Spend | Purchase not budgeted for; "something came up" |

## Family E: Social & Relational Life

## Family F: Emotional & Physical Experience

| Code | Label | When to Apply |
|:---:|:---|:---|
| F01 | Morning Energy/Optimism | Positive, energised start: "happy", "energetic", "relaxed", "peaceful" |
| F02 | Stress Peak | Explicit or strongly implied stress at any point in the day |
| F03 | Frustration | Blocked goal, failed task, system failure causing frustration |
| F04 | Relief/Accomplishment | Positive resolution after effort: "fruitful", "productive", "satisfied" |
| F05 | Resignation/Adaptation | Acceptance of difficulty as normal: "this is how it is", normalised frustration |
| F06 | Evening Decompression | Relaxation, entertainment, or wind-down in the evening |
| F07 | Anxiety | Worry about finances, safety, relationships, or the future |
| F08 | Physical Health Barrier | Sickness, fatigue, hunger, or physical condition constraining the day: "sick", "tired", "running stomach" |

## Family I: Aspirations & Unmet Needs

| Code | Label | When to Apply |
|:---:|:---|:---|
| I01 | Unmet Task (Could Not Do) | Explicitly states something wanted to do but could not: "I wanted to... but..." |
| I02 | Workaround Described | Describes alternative strategy when primary route failed: "I walked instead", "used someone else's phone" |
| I03 | Opportunity Awareness | Describes a moment where better connectivity/service would have created value or income |
| I04 | Service Expectation vs. Reality Gap | Disappointment that something didn't work as expected: app, network, service, product unavailable |
| I05 | Community/Social Aspiration | Career development, helping community, making opportunities for youth, growth language |

## Family J: Power & Infrastructure

| Code | Label | When to Apply |
